When Jesse asks to sort, triage, clean, filter, or organize emails in ANY form, you MUST:

**Step 1 — Extract parameters from the directive:**
- Count: "top 100" → 100 | "last 28" → 28 | "all" / "everything" → all | bare number → use it
- Time: "past 2 hours" → --since <epoch> | "today" → --since <midnight epoch>
- When ambiguous, default to --total 50

**Step 2 — Call the triage script:**
Use the exec tool to run:
```
/Users/jesseharper/Documents/Workshop/workhorse/scripts/harper_email_triage.sh <ARGS>
```
Where <ARGS> is the output of:
```
/Users/jesseharper/Documents/Workshop/workhorse/scripts/harper_parse_directive.sh "<original directive>"
```

**Examples of valid directives and what they map to:**
- "sort through top 100 emails" → `--total 100 --batches 10 --approve-all`
- "filter the last 28 emails" → `--total 28 --batches 3 --approve-all`
- "go through emails from the past 2 hours" → `--since <epoch> --batches 5 --approve-all`
- "sort all unread mail" → `--total all --batches 20 --approve-all`
- "do 50 emails" → `--total 50 --batches 5 --approve-all`

**Never ask "how many?" — parse the directive and proceed. The user will tell you if something is wrong.**

After the mission completes, a SMS summary is automatically sent. Just confirm to the user that the mission is running.
